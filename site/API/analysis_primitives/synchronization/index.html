Synchronization

Phase synchronization between paired motion signals.

This module provides a real-time estimator of phase synchronization between two signals using the Phase Locking Value (PLV). PLV measures the consistency of the instantaneous phase difference between two oscillatory signals over time.

A PLV of 1.0 means the two signals are perfectly phase-locked; a value near 0.0 indicates no systematic phase relationship.

Typical use cases include:

  • Quantifying interpersonal synchrony between two performers.
  • Detecting coordination between body segments.
  • Analysing entrainment in musical/dance performance contexts.
References

Lachaux, J.-P., Rodriguez, E., Martinerie, J., & Varela, F. J. (1999). Measuring phase synchrony in brain signals. Human Brain Mapping, 8(4), 194–208.

Real-time phase synchronization analyser for paired signals.

Computes the Phase Locking Value (PLV) between the first two signals present in the input window. An optional bandpass filter can be applied before phase extraction to focus on a specific frequency band of interest.

Read more in the User Guide.

Parameters:
  • filter_params (tuple of (float, float, float) or None, default: None ) –

    Bandpass filter specification as (lowcut_hz, highcut_hz, fs_hz). When None (default) the raw signal is used without filtering.

Examples:

>>> sync = Synchronization()                        # no bandpass filter
>>> sync = Synchronization(filter_params=(0.5, 4.0, 50.0))  # 0.5–4 Hz band
Raises:
  • ValueError –

    If filter_params has the wrong number of elements or invalid frequency values (e.g. highcut >= Nyquist).

class Synchronization(DynamicFeature):
    """Real-time phase synchronization analyser for paired signals.

    Computes the **Phase Locking Value (PLV)** between the first two signals
    present in the input window.  An optional bandpass filter can be applied
    before phase extraction to focus on a specific frequency band of interest.

    Read more in the [User Guide](../../user_guide/theoretical_framework/analysis_primitives/synchronization.md).

    Parameters
    ----------
    filter_params : tuple of (float, float, float) or None, optional
        Bandpass filter specification as `(lowcut_hz, highcut_hz, fs_hz)`.
        When `None` (default) the raw signal is used without filtering.

    Examples
    --------
    >>> sync = Synchronization()                        # no bandpass filter
    >>> sync = Synchronization(filter_params=(0.5, 4.0, 50.0))  # 0.5–4 Hz band

    Raises
    ------
    ValueError
        If `filter_params` has the wrong number of elements or invalid
        frequency values (e.g. `highcut >= Nyquist`).
    """

    def __init__(self, filter_params=None):
        super().__init__()
        self.filter_params = filter_params

    @property
    def filter_params(self):
        """Bandpass filter parameters `(lowcut, highcut, fs)` or `None`."""
        return self._filter_params

    @filter_params.setter
    def filter_params(self, value):
        self._filter_params = validate_and_normalize_filter_params(value)

    def compute(self, window_data: np.ndarray) -> SynchronizationResult:
        """Compute phase synchronization (PLV) between the first two signals.

        Parameters
        ----------
        window_data : numpy.ndarray of shape (Time, N_signals) or (Time, N_signals, N_dims)
            Window of motion data.  The array is flattened to `(Time, N_signals)`
            before processing.  At least two signals must be present.

        Returns
        -------
        SynchronizationResult
            Result containing the PLV.  Returns
            `SynchronizationResult(is_valid=False)` when fewer than 2 signals
            are available.

        Notes
        -----
        The PLV is computed via the Hilbert transform applied to the (optionally
        filtered) signal pair.  See
        [compute_phase_synchronization][pyeyesweb.utils.signal_processing.compute_phase_synchronization]
        for implementation details.
        """
        # Flatten to (Time, Signals)
        data = window_data.reshape(window_data.shape[0], -1)

        if data.shape[1] < 2:
            return SynchronizationResult(is_valid=False)

        # Assumes compute_phase_synchronization expects a 2D array of (Time, N_Signals)
        plv = compute_phase_synchronization(data, self.filter_params)
        return SynchronizationResult(plv=float(plv))
